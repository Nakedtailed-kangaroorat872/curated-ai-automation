# Contributing to Curated AI Automation

Thank you for your interest in contributing! This document provides guidelines for adding new tools or improving this curated list.

## How to Contribute

### Adding a New Tool

1. **Fork** this repository
2. **Create a new branch** for your addition: `git checkout -b add-tool-name`
3. **Add the tool** to the appropriate category in `README.md`
4. **Follow the format** shown below
5. **Submit a pull request** with a clear description

### Entry Format

Each entry should follow this table format:

```markdown
| [Tool Name](https://tool-url.com) | Brief description ending with a period. | Type | [domain.com](https://tool-url.com) |
```

- **Tool Name**: Official name with link to homepage or GitHub repo
- **Description**: One sentence, max 120 characters, ending with a period
- **Type**: Either `Open Source` or `Commercial`
- **Link**: Short domain link for readability

### Criteria for Inclusion

Your submission must meet ALL of the following:

- [ ] Tool is actively maintained (updated within the last 12 months)
- [ ] Has a working website or repository
- [ ] Open-source tools must have at least 100 GitHub stars
- [ ] Commercial tools must have a functional product (not just a landing page)
- [ ] Tool has an AI or automation focus
- [ ] Not a duplicate of an existing entry

### Categories

Add your tool to the most relevant category:

- **Workflow Platforms** -- General-purpose automation connecting apps and services
- **AI-Native Automation** -- Platforms with AI/LLM at the core
- **Chatbot Builders** -- Conversational AI and chatbot platforms
- **RPA** -- Robotic process automation tools
- **Email & Marketing Automation** -- Email outreach and marketing tools
- **Data Automation** -- Data pipelines, ETL, and data quality
- **Document Processing** -- Document extraction and classification
- **Browser Automation** -- Web automation, testing, and scraping
- **Voice & Phone Automation** -- Voice AI and telephony

If your tool does not fit any category, propose a new one in your PR description.

### Suggesting a New Category

If you believe a new category is needed:

1. Open an [issue](https://github.com/awesome-ai-tools/curated-ai-automation/issues/new?template=new-category.yml) first
2. Include at least 5 tools that would belong in the category
3. Explain why existing categories do not fit

## Reporting Issues

### Broken Links

Use the [broken link template](https://github.com/awesome-ai-tools/curated-ai-automation/issues/new?template=broken-link.yml) to report dead links.

### Incorrect Information

Open a general issue describing what is wrong and what the correct information should be.

## Code of Conduct

- Be respectful and constructive
- No self-promotion spam -- tools must genuinely fit the criteria
- One tool per pull request for easier review
- Do not remove existing entries without justification

## Review Process

1. A maintainer will review your PR within 7 days
2. We may request changes or additional information
3. Once approved, your contribution will be merged

Thank you for helping make this list better!
